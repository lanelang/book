# 类型

类型是 Lane 编程语言中的一个极其重要的概念。数据的组织、底层表示和行为抽象等都离不开类型。

## 内置类型

Lane 语言内置了几种重要类型：

```
Int
Double
String
Unit
Bool
Void
```

这些类型的定义不向程序员开放。不过，我们可以大致将 `Void`、`Bool` 和 `Unit` 理解为如下定义：

```
enum Void {}
enum Bool {
  true()
  false()
}
enum Unit {
  unit()
}
```

`Void` 是无法构造的；`Unit` 总是只有一个值 `unit()`；`Bool` 有两个值 `true()` 和 `false()`。

在实际的 Lane 程序中，无需导入上述类型的定义；`unit()`、`true()` 和 `false()` 的实际写法分别是 `()`、`true` 和 `false`。

整数类型 `Int` 是 64 位整数，涵盖从 `-(2^63)` 到 `2^63 - 1` 的所有整数。

浮点数类型 `Double` 是符合 IEEE 754 标准的 64 位浮点数，涵盖正无穷 `double_inf`、负无穷 `double_neg_inf` 和非数 `double_nan`。

Lane 的字符串类型是 ASCII 码的序列。

## 函数类型

函数类型用箭头表示：`(A) -> B` 表示接收一个 `A` 类型的参数并返回一个 `B` 类型的值。例如：

```
fn f(x : Int) -> Int {
  x * 2
}
```

它的类型是：

```
f : (Int) -> Int
```

对于一个类型为 `(A) -> B` 的函数 `f` 和一个类型为 `A` 的值 `x`，表达式 `f(x)` 的类型一定是 `B`。

函数的参数个数称为函数的元数（arity）。下面是一个二元函数的例子：

```
fn f(x : Int, y : Int) -> Int {
  x + y
}
```

函数也可以是零元的，例如：

```
fn f() -> Int {
  42
}
```

如果函数 `f` 的参数中包含函数，那么 `f` 就是高阶函数；如果它的参数中不包含函数，那么它就是一阶函数。普通值可以看作零阶的。如果 `f` 的所有参数中阶数最高的是 `n` 阶，那么 `f` 是 `n + 1` 阶函数。

例如，下面这个函数是二阶的：

```
fn[A, B] apply(f : (A) -> B, x : A) -> B {
  f(x)
}
```

函数类型也可以携带类型变量。例如，上述函数 `apply` 的类型可以表示为：

```
apply : [A, B]((A) -> B, A) -> B
```

这个类型可以读作：如果 `A`、`B` 是类型，`f` 的类型是 `(A) -> B`，`x` 的类型是 `A`，那么 `apply[A, B](f, x)` 的类型就是 `B`。

这里的类型参数通常可以由编译器自动推导，因此不必总是写出。通常，`apply[A, B](f, x)` 可以直接写作 `apply(f, x)`。

对于上面这个函数的类型，也可以只读一半：如果 `A`、`B` 是类型，那么 `apply[A, B]` 的类型是 `((A) -> B, A) -> B`。

这种理解方式对于下面要讨论的存在类型非常必要。

## 存在类型

我们已经见过泛型的例子：列表类型 `List`。它可以写作 `type List : [Type] -> Type`，读作：如果 `A` 是一个类型，那么 `List[A]` 也是一个类型。也就是说，`List` 本身不是类型。

可以定义一个最简单的泛型类型 `Box`，它只用于存储一个值：

```
enum Box[T] {
  box(T)
}
```

可以认为，`box` 的类型是 `[T](T) -> Box[T]`，这种常见的多态类型称为全称类型。但有时我们需要一个能够存储任意内容的盒子 `Hide`，却不希望存储内容的类型反映在盒子本身的类型上。换言之，我们需要一个类型为 `[T](T) -> Hide` 的构造器，以及相应的 `Hide` 类型定义。在 Lane 中，可以这样写：

```
enum Hide {
  hide[T](T)
}
```

注意，此时类型变量 `T` 不再由枚举类型声明引入，而是由构造器 `hide` 引入。在这个例子中，`Hide` 本身就是一个类型，即 `Hide : Type`；而 `hide` 需要带上类型参数，类型为 `[T](T) -> Hide`。它与 `box` 的类型左侧相同，仅右侧不同，但本质已经发生了变化。

如何使用这样的类型？可以使用模式匹配，或者使用专门处理不可拒绝模式的绑定：

```
let v : Hide = hide[Int](10)
...
let hide[T](x) = v
// or
match v {
  hide[T](x) => ...
}
```

但要注意：由于 `v` 的类型是 `Hide`，使用它时并不知道其中保存的是一个整数，只知道解包后 `x` 的类型是 `T`。因此，值 `x` 和类型 `T` 都不能逃出其作用域。例如，下面的程序是非法的：

```
fn[T] escape(v : Hide) -> T {
  let hide[T](x) = v
  x
}
```

因为对 `v` 的解包操作本质上是将隐藏的类型匹配到类型变量 `T`，这个 `T` 遮蔽了上下文中由函数类型参数引入的类型变量 `T`。因此，上述程序与下面的程序并无二致：

```
fn[R] escape(v : Hide) -> R {
  let hide[T](x) = v
  x
}
```

这样就能看出问题所在：`x` 的类型是 `T`，而函数要求返回 `R`。显然，没有任何证据表明这两个类型相同，因此该函数无法通过编译。
